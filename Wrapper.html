Wrapper Class

A Wrapper class in Java converts a primitive data type into an object.
Java is an object-oriented language, Java collections and other frameworks work only with objects, so wrapper classes help us use primitive values as objects.

Why do we need Wrapper Classes?

  • Collections (ArrayList, HashMap) cannot store primitive data, only objects.
  • Provides many useful methods (parsing, comparing).
  • Needed for serialization.
  • Supports autoboxing/unboxing.
  • Helps in method parameter passing when objects are required.

Creating Wrapper Objects

Using constructors (old method, not recommended now):

Integer obj = new Integer(10);

Using valueOf() (recommended):

Integer obj = Integer.valueOf(10);

AutoBoxing (Primitive → Object Automatically)

automatic conversion of a primitive data type into its wrapper object.

Example: 
int a = 10;
Integer obj = a;   // AutoBoxing

No need to manually create an object.

Real use: 
ArrayList<.Integer.> list = new ArrayList<>();
list.add(10);  // AutoBoxing: int → Integer

Unboxing (Object → Primitive Automatically)

Conversion from wrapper object back to primitive type.

Example: 
Integer obj = 100;
int x = obj;   // Unboxing

Earlier this would require:

int x = obj.intValue();

Wrapper Class Utility Methods

Examples using Integer:

parseXxx()

Convert string into primitive.

int n = Integer.parseInt("123");

valueOf()

Convert primitive or string to wrapper object.

Integer i = Integer.valueOf("50");

toString()

Convert number to string.

String s = Integer.toString(200);

Wrapper Class Example Program

class WrapperExample {
public static void main(String[] args) {
int a = 10;
    // AutoBoxing
    Integer obj = a;

    // Unboxing
    int b = obj;

    System.out.println("Primitive value: " + a);
    System.out.println("Wrapper value: " + obj);
    System.out.println("Unboxed value: " + b);
}
}
Output: 
Primitive value: 10
Wrapper value: 10
Unboxed value: 10

Example: Using Wrapper with Collections

ArrayList cannot store primitives directly.

import java.util.*;
class WrapperWithCollection {
public static void main(String[] args) {
ArrayList list = new ArrayList<>();
    list.add(10);   // AutoBoxing
    list.add(20);
    list.add(30);

    for(Integer n : list) {
        System.out.println(n);
    }
}
}
Output: 
10
20
30

Real-Life Example Using Wrapper Classes

import java.util.ArrayList;

public class WrapperExample {
    public static void main(String[] args) {
        ArrayList<.Integer.> list = new ArrayList<>(); 

        list.add(10);  // autoboxing: int → Integer
        list.add(20);

        int value = list.get(0);  // unboxing: Integer → int

        System.out.println("Value: " + value);
    }
}
Output: 
Value: 10

When Should You Use Wrapper Classes.

  • When using Collections
  • When you need to store null values
  • When using class methods (parse, valueOf, compare)
  • When working with generics

Some extra Points

1. All wrapper Classes are Immutable (cannot be changed after creation).

Integer a = 10;
a = 20; 			// new object created.

2. Wrapper Classes are Final (Inheritance Not Allowed) -not extend them.

3. Wrapper Classes are stored in HEAP memory.

Primitive values store in stack,
Wrapper objects store in heap.

4. Wrapper Class Provides CONSTANTS - Useful predefined value

Integer.MIN_VALUE	-minimum value of int
Integer.MAX_VALUE	-mazmim value if int
Character.SIZE		-char size in bits

5. Utility Methods

 parseInt() – convert String → int
 valueOf() – convert String → wrapper object
 toString() – convert object → String
 compare() – compare values
 isDigit() – check digit

6. Wrapper Object can be NULL

Primitive types (int, boolean) cannot store null.
But wrapper classes can store null.
Example: 
   Integer a = null; // valid
       int b = null;     // error

7. Wrapper Classes are used in GENERICS

(One of the biggest reasons)

ArrayList<.int.>		 // not valid
ArrayList<.Integer.>	//must

8. Wrapper Classes Support Type Conversion

Example: 
Integer a = Integer.valueOf("100");
int b = a.intValue();
double c = a.doubleValue();
Wrapper classes provide many methods to convert primitive value.

9.Wrapper Classes Help in BOXING & UNBOXING

  • Boxing = primitive → object
  • Unboxing = object → primitive

10. Wrapper Classes are Part of java.lang Package

No need to import wrapper classes because they already available in java.lang package.

11. Wrapper Classes inside Collection Framework

ArrayList<.Integer.> list = new ArrayList<>();
list.add(10); // autoboxing

Work with Collections wrapper class is compulsory.

12. Wrapper Classes Used for Serialization

Primitive not serialize but wrapper objects serializable
