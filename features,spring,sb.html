Java 8 Features

1. Lambda Expressions

  • Introduced functional programming in Java
  • Lets you write shorter, cleaner code

Syntax:

(parameter) -> expression

Example:

List list = Arrays.asList("A", "B", "C");
list.forEach(item -> System.out.println(item));

2. Functional Interfaces

  • An interface with only one abstract method
  • Example: Runnable, Callable, Comparable, Predicate, Function, Supplier, Consumer

Example:

@FunctionalInterface
interface Demo {
    void show();
}

3. Default & Static Methods in Interfaces

• Interfaces can now have method implementations.

Example:

interface A {
    default void display() { System.out.println("Hello"); }
    static void test() { System.out.println("Static method"); }
}

4. Stream API

Used for processing collections in a functional + efficient way.

Common operations

  • filter()
  • map()
  • sorted()
  • collect()

Example:

List nums = Arrays.asList(1,2,3,4,5);
List even = nums.stream()
                         .filter(n -> n % 2 == 0)
                         .collect(Collectors.toList());

5. Optional Class

  • Avoids NullPointerException
  • Used to handle nullable values elegantly

Example:

Optional name = Optional.ofNullable(null);
System.out.println(name.orElse("Default"));

6. Method References

Short form of lambda expressions.

Types:

  • Class::staticMethod
  • object::instanceMethod
  • Class::new

Example:

list.forEach(System.out::println);

7. Date and Time API (java.time)

A new improved package replacing old Date and Calendar.

Common classes:

  • LocalDate
  • LocalTime
  • LocalDateTime
  • ZonedDateTime
  • Period, Duration

Example:

LocalDate date = LocalDate.now();

8. Nashorn JavaScript Engine

Allows execution of JavaScript code inside Java (Mostly replaced now)

9. CompletableFuture

  • Advanced asynchronous programming
  • Handles non-blocking code easily

Example:

CompletableFuture.supplyAsync(() -> "Hello");

What is Spring?

Spring is a Java framework used to build backend applications.
It helps developers create applications that are:

  • Easy to manage
  • Secure
  • Fast to develop
  • Well-structured

Why was Spring created?

Before Spring, Java applications were difficult to manage because everything was tightly connected. Spring introduced loose coupling, meaning parts of the program can work independently and are easier to maintain.

Important Concepts in Spring

1 IoC (Inversion of Control)
Instead of you creating objects, Spring creates and manages them.
This makes the application cleaner and easier to control.
2 DI (Dependency Injection)
If class A needs class B, Spring will provide it automatically.
No need to manually create objects.
3 Modules
Spring has many modules like:
  • Spring Core
  • Spring MVC
  • Spring JDBC
  • Spring Security

Where is Spring used?

• Web applications
• Backend systems
• Enterprise applications
• Microservices
• Banking and financial systems

What is Spring Boot?

Spring Boot is Spring but easier and faster.
With Spring, you had to configure many things manually.
Spring Boot removes that difficulty by providing: 

 Auto configuration
 Built-in web server
 Quick project setup
 Production-ready features

Example:

With Spring, you needed many configuration files.
With Spring Boot, you can start a web app by writing just a few lines of code. 

Why do people use Spring Boot?
1️. Faster Development
Less setup, more coding.
2️.  Embedded Server
Tomcat/Jetty already included.
Just run the application — no separate server installation required.
3️.  Production Ready
Features like:
•	Health monitoring
•	Metrics
•	Logging
•	Error handling
are already built-in.
4️.  Popular for Microservices
Netflix, Amazon and many companies use Spring Boot for microservices.

Difference:

Feature Spring Spring Boot
Setup Manual and more configuration Auto-configuration
Server Needs external server Built-in Tomcat server
Development Speed Slower Faster
Complexity More Less, beginner-friendly
• Spring is the big Java framework for building applications.
• Spring Boot is the easy and fast version of Spring with auto-configuration.
