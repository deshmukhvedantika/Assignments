Hoisting

       Hoisting means variables and functions put on the top before js code running.Declaration write below of the output line.

Example:            console.log(a);
                    var a = 10;

js internally do like this 				Output:  undefined
			var a;
			console.log(a);
                        var a = 10;

•	var:           goes to the top and gives undefined output.
•	let/const:     go to the top but cannot be used  before declaration  they throws an error.
•	function:      entire function goes to top and can be used.

 let
1 . Block scoped – work only inside the block.
    {
        let a =10;
    }
    console.log(a);                                  			//error

2 . Cannot be re-declared in the same block.
	let x=5;
	let x=10;
3 . Value can be updated but variable cannot be re-declared.
	let x=5;
        x=10;                                      			 //allowed

4 . Hoisted,but in the (TDZ)temporal dead zone using before declaration –they throw an error.
Goes to top (hoist) but not used gives error.
    console.log(a);                                  			//error
    let a=10;

var- old
1 . Not block scope .
Even if you decared inside block it can be accessible outside from outside.
    {
        var x =10;
    }
	console.log(x);               			                  //works

2 . Can be re-declared   -this is biggest problem
   	 var a =5;
    	 var a =10;					                  //allowed

3 . Hoisting is possible and gives undefined value.
	console.log(a);					                  //undefined
	var a =10;
